# Roadmap v3 Changes: Strategic Restructure

## Core Changes from v2

1. **Data foundation first** — verified trading data before AI features
2. **Prop Shield as paid wedge** — prop risk protection monetizable earlier than coaching
3. **Trader Workspace as explicit layer** — journal, analytics, behavior metrics
4. **RAG/coaching deferred** — from Phase 1 to Phase 6
5. **NeuroFund Elite added** — compliance-safe rewards/eligibility program
6. **Edge Forensics added** — losses → testable hypotheses
7. **NeuroScore added** — risk-adjusted trader ranking
8. **Track C eliminated** — compliance embedded per-phase

## Revised Phase Structure

| Phase | Theme | Goal |
|---|---|---|
| **Phase 0** | Marketing + Demo | Validate market, build waitlist |
| **Phase 1** | Trading Data Foundation | Normalized broker/trade data |
| **Phase 2** | Trader Workspace | Journal + analytics + behavior metrics |
| **Phase 3** | Prop Shield | **FIRST REVENUE:** prop risk + Tradovate protection |
| **Phase 4** | ICT Event Intelligence | Programmable ICT events |
| **Phase 5** | EdgeLab Core | Event-driven backtesting |
| **Phase 6** | AI Trade Review + RAG | Grounded coaching with citations |
| **Phase 7** | Edge Forensics | Losses → testable hypotheses |
| **Phase 8** | NeuroScore + Leaderboard | Risk-adjusted trader ranking |
| **Phase 9** | NeuroFund Elite Rewards | Company-sponsored rewards, eligibility |
| **Phase 10** | Allocation Watchlist | Company-sponsored allocation pipeline |
| **Phase 11** | Advanced ML Research | NSLM, hybrid models, NeuroQuant, NeuroTrader |

## NeuroFund Compliance Framework

### Approved Language

- "Company-sponsored trader allocation and rewards program funded from NeuroSpect revenue"
- "Premium members may become eligible for company-sponsored rewards, challenge reimbursements, grants, and future internal allocation consideration"
- "NeuroSpect may select qualified traders for challenge sponsorships"
- "May allocate", "may be eligible for", "consideration"

### Forbidden Language

- "Pooled user capital" / "subscriber money invested" / "investment pool"
- "Fund returns" / "members receive returns"
- "Your subscription is invested"
- "Community trading pool"
- "Guaranteed rewards" / "guaranteed returns"
- "Copy top traders" (without separate legal review)
- "Subscribers fund traders"

### Framing Rule

NeuroFund should be framed as a **premium eligibility / company-sponsored rewards and allocation program**, not a subscriber-funded investment pool. The rewards reserve is funded from company revenue, using "may allocate" language instead of implying user money is invested.

## Strategic Scoring Priorities

1. Prioritize the **paid wedge**: Tradovate + prop risk protection + tilt lockouts + advanced analytics
2. Prioritize **proprietary data loops**: verified trade data → AI review → edge forensics → testable features → improved strategy
3. Prioritize **ICT-specific infrastructure**: FVGs, sweeps, order blocks, sessions, liquidity events
4. **Reframe NeuroFund** as rewards, sponsorships, verified leaderboard, eligibility
5. **Defer** anything requiring pooled user capital, subscriber returns, managed accounts, fund operations

## Build Order Principle

```
verified data → risk engine → event engine → backtesting → AI review → edge forensics → scoring → rewards/allocation eligibility → advanced ML
```

This creates a defensible platform instead of a pile of disconnected AI features.
