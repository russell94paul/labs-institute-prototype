# NeuroSpect Roadmap

Single source of truth for project state: phases, status, boot prompts, deviations, and the master plan.

## Structure

```
roadmap/
├── CLAUDE.md                        # This file
├── plan.md                          # Active master plan (v2 — EdgeLab architecture)
├── status.md                        # Dashboard: all phases, current status, blockers
└── phases/
    └── phase-N-name/
        ├── README.md                # Canonical: goals, scope, exit criteria, status
        ├── deviations.md            # What changed from plan during implementation
        └── boot-prompts/
            ├── planning.md          # Boot prompt for planning/design sessions
            └── execution.md         # Boot prompt for implementation sessions
```

Plan version history lives in `initial-plan/` (read-only archive). See `initial-plan/CLAUDE.md` for the version table.

## Boot Prompts

Each phase has two boot prompts in `boot-prompts/`:

- **`planning.md`** — Load this when starting a planning/design session for the phase. Includes phase goals, scope, constraints, prior decisions, and open questions. Used before implementation begins.
- **`execution.md`** — Load this when starting an implementation session. Includes what to build, key files to modify, technical decisions already made, active tickets, and deviations from previous phases.

Boot prompts are **generated by `/sync`**, not hand-written. They are assembled from:
1. Phase definition (this README)
2. Actual code state (git log, file structure)
3. Previous phase deviations (what was actually built vs planned)
4. Active Linear tickets (status, assignments, blockers)

Complex tickets that span multiple sessions may also have their own boot prompt in the personal wiki ticket folder.

## Deviations

When implementation diverges from the plan, capture it in `deviations.md`:

```markdown
## Deviation: Used pgvector instead of Chroma
- **Planned:** Evaluate Chroma, Pinecone, pgvector
- **Actual:** Went straight to pgvector — corpus too small to justify separate DB
- **Impact on future phases:** Phase 2 assumes pgvector, not a separate vector service
- **Date:** 2026-05-15
```

`/sync` reads deviations and injects them into downstream boot prompts automatically.

## Relationship to Other Wikis

- `roadmap/` = **canonical project state** (what phases exist, their status, boot prompts, deviations)
- `wiki/` = **canonical team knowledge** (ICT concepts, architecture, course content)
- `vlad-wiki/`, `paul-wiki/` = **personal working memory** (tickets, implementation notes, research drafts)

Personal wikis reference `roadmap/` phases but do not duplicate phase definitions. They hold ticket-level detail and personal notes.
