# Prompts

Engineering and product prompts used to build and operate NeuroSpect. Separate from the ICT knowledge base (`wiki/`) and the deployed coaching prompts (`api/app/coach/prompts/`).

## Structure

```
prompts/
├── CLAUDE.md              # This file
├── index.md               # Registry of all prompts
├── phases/                # Prompts tied to specific roadmap phases
│   ├── phase-0/
│   ├── phase-1/
│   └── ...
├── ad-hoc/                # One-off prompts not tied to a phase
└── meta/                  # Meta-prompts (prompts about prompts, prompt engineering)
```

## Conventions

### Frontmatter

Every prompt file uses YAML frontmatter:

```yaml
---
version: 1
phase: 1                          # omit for ad-hoc/meta
purpose: Short description
status: active | superseded | experimental
supersedes: phases/phase-0/foo-v1.md  # if this replaces an earlier version
model: sonnet | opus | haiku      # which model this was designed for (optional)
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

### File naming

`<descriptive-slug>-v<N>.md` — e.g., `rag-retrieval-design-v2.md`. Bump the version number and add `supersedes:` when creating a new version rather than editing in place.

### Status lifecycle

- **experimental** — draft, untested or partially tested
- **active** — current version in use
- **superseded** — replaced by a newer version (link via `supersedes:` in the successor)

### What goes here vs elsewhere

| Content | Location |
|---|---|
| Engineering/build prompts | `prompts/phases/` or `prompts/ad-hoc/` |
| Meta-prompts (prompt engineering) | `prompts/meta/` |
| Deployed coaching system prompt | `wiki/concepts/ai-coach/` (canonical) → `api/app/coach/prompts/` (bundled) |
| Boot prompts (session context) | `roadmap/phases/phase-N/boot-prompts/` (auto-generated by `/sync`) |
| Personal scratch prompts | `paul-wiki/scratch/` or `vlad-wiki/scratch/` |

## Phase 4 Graduation

This folder is the informal, filesystem-based version of the prompt versioning system. In Phase 4, it graduates to a Python module with database-backed registry, A/B testing, regression detection, and audit trails. See `paul-wiki/components/prompt-versioning-module.md` for the full design spec.
