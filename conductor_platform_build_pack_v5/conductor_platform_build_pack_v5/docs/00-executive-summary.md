# Executive Summary

Conductor should be built as a platform-first product that can build, modify, migrate, operate, and improve apps/products across domains.

## Primary use cases

1. **New Product Build** — idea to market-ready roadmap and implementation.
2. **Existing Product Improvement** — audit an app/repo and recommend refactor, feature extension, partial rebuild, full rebuild, or hybrid migration.
3. **Data Platform / Service Modernization** — modernize existing data/analytics clients while preserving current functionality.

## Platform modules

- Build Studio
- Product Onboarding Studio
- Guided Product Decision Simulator
- Client Product Portal
- Trust-Aware Discovery & Access Manager
- Data Platform Modernization Studio
- Market Intelligence + Growth Engine
- Context + Memory Engine
- Infrastructure / Deployment / Operations
- Self-Improvement Engine

## Key rule

Conductor should request the minimum access required to make the next decision safely. It should not require full data, credentials, or infrastructure access upfront.
