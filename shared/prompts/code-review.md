## Code Review Checklist

Review for:

1. **Critical** — bugs, security vulnerabilities, data loss risks, broken logic
2. **Warnings** — missing error handling, edge cases, race conditions, performance
3. **Patterns** — project conventions, naming consistency, DRY violations
4. **Tests** — are new features tested? Edge cases covered?

For each finding: severity, file, issue, recommended fix.
End with verdict: APPROVE, REQUEST CHANGES, or NEEDS DISCUSSION.
